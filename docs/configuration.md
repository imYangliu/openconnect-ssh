# OCH 配置文件说明

OCH 的主配置文件是 TOML：

```text
~/.config/och/config.toml
```

macOS GUI 会读写这份文件；CLI 的 `och` 和 `och-vpn` 也会读取它。可以用 `OCH_CONFIG_FILE` 指向其他 TOML 文件：

```bash
OCH_CONFIG_FILE=/path/to/config.toml och och-target
OCH_CONFIG_FILE=/path/to/config.toml och-vpn connect
```

## 读取顺序

`och` 和 `och-vpn` 都先读取 `OCH_CONFIG_FILE` 指向的 TOML 文件，默认是 `~/.config/och/config.toml`。

随后读取一个 env 覆盖文件：

1. 如果设置了 `ENV_FILE`，读取 `ENV_FILE`
2. 否则如果当前目录有 `.env`，读取当前目录 `.env`
3. 否则如果启动脚本设置了 `PROJECT_ENV_FILE`，读取 `PROJECT_ENV_FILE`

覆盖文件在 TOML 之后加载，所以同名变量会覆盖 TOML 解析出的值。启动命令前已经设置的非空环境变量也会优先于 TOML；例如 `VPN_HOST=other.example.com och-vpn status` 不会被 TOML 中的 `[vpn].host` 覆盖。

`~/.config/och/och.env` 和 `~/.config/och/och-vpn.env` 不是当前脚本的默认读取路径；旧示例文件只适合迁移参考。新配置请优先写 `config.toml`，密码和其他私密值写 `.env` 或 `ENV_FILE`。

## 完整示例

```toml
# Generated by OCH. VPN password is stored in Keychain, not in this file.

[vpn]
host = "vpn.example.com"
user = "your_vpn_username"
auth_group = ""

[ssh]
host = "och-target"
target_host = "your-campus-host.example"
user = "your-ssh-user"
port = "22"

[routes]
extra = ["10.0.0.0/8", "192.168.0.0/16"]

[proxy]
local_host = "127.0.0.1"
local_port = "7890"
remote_port = "7890"

[paths]
och = "/opt/homebrew/bin/och"
och_vpn = "/opt/homebrew/bin/och-vpn"
askpass = "/opt/homebrew/libexec/och/och-sudo-askpass.sh"
```

## 字段说明

### `[vpn]`

| 字段 | 对应变量 | 必填 | 说明 |
| --- | --- | --- | --- |
| `host` | `VPN_HOST` | 是 | VPN 网关地址，传给 `openconnect`。 |
| `user` | `VPN_USER` | 是 | VPN 用户名。 |
| `auth_group` | `VPN_AUTHGROUP` | 否 | 认证组/realm；网关不需要时留空。 |

VPN 密码不要写入 TOML。GUI 使用 Keychain 保存密码；CLI 推荐在本机 `.env` 或 `ENV_FILE` 指向的私有覆盖文件中设置 `VPN_PASSWORD`，也可以在连接时临时输入。

### `[ssh]`

| 字段 | 对应变量 | 必填 | 说明 |
| --- | --- | --- | --- |
| `host` | `DEFAULT_HOST` | 否 | `och` 未传目标主机时使用的默认 SSH Host。 |
| `target_host` | `TARGET_HOST` | `verify`/`ssh` 需要 | 真实目标主机；也用于连接后验证路由和端口。 |
| `user` | `TARGET_SSH_USER` | 否 | SSH 用户；默认是当前系统用户。 |
| `port` | `TARGET_PORT` | 否 | SSH 端口；默认 `22`。 |

GUI 还会用这些字段生成 `~/.ssh/och.config` 中的托管 Host。

### `[routes]`

| 字段 | 对应变量 | 必填 | 说明 |
| --- | --- | --- | --- |
| `extra` | `MACOS_EXTRA_ROUTES` / `VPN_ROUTES` | 视平台而定 | CIDR 数组，例如 `["10.0.0.0/8"]`。 |

macOS 默认使用 OpenConnect 自带的 `vpnc-script`，通常可以留空。需要额外让某些网段走 VPN 时填写 `extra`。

Linux/WSL 分流通常需要 `VPN_ROUTES`。TOML 解析时会把 `routes.extra` 同时映射到 `MACOS_EXTRA_ROUTES` 和 `VPN_ROUTES`，因此同一份配置可跨平台复用；如果需要更复杂的分流脚本，可以在 env 覆盖文件里设置 `VPN_SCRIPT_CMD`。

### `[proxy]`

| 字段 | 对应变量 | 必填 | 说明 |
| --- | --- | --- | --- |
| `local_host` | `PROXY_LOCAL_HOST` | 否 | `och --proxy` 映射到的本地地址，默认 `127.0.0.1`。 |
| `local_port` | `PROXY_LOCAL_PORT` | 否 | 本地端口，默认 `7890`。 |
| `remote_port` | `PROXY_REMOTE_PORT` | 否 | 远端暴露端口，默认 `7890`。 |

`och --proxy och-target` 会追加 SSH 反向映射：

```text
远端 remote_port -> 本地 local_host:local_port
```

### `[paths]`

| 字段 | 对应变量 | 必填 | 说明 |
| --- | --- | --- | --- |
| `och` | `OCH_PATH` | 否 | GUI 生成 SSH `ProxyCommand` 时使用的 `och` 路径。 |
| `och_vpn` | `OCH_VPN_PATH` / `CONNECT_SCRIPT` | 否 | `och` 调用 VPN 脚本时使用的路径。 |
| `askpass` | `OCH_ASKPASS_PATH` / `SUDO_ASKPASS` | 否 | GUI 触发 `sudo -A` 时使用的 askpass 脚本。 |

Homebrew 默认路径通常是：

```toml
[paths]
och = "/opt/homebrew/bin/och"
och_vpn = "/opt/homebrew/bin/och-vpn"
askpass = "/opt/homebrew/libexec/och/och-sudo-askpass.sh"
```

Intel macOS 或 `/usr/local` 安装时，把路径改成对应前缀即可。

## Env 覆盖文件

`.env` 或 `ENV_FILE` 使用 shell 变量语法：

```bash
VPN_HOST=vpn.example.com
VPN_USER=your_vpn_username
TARGET_HOST=your-campus-host.example
TARGET_PORT=22
TARGET_SSH_USER=your-ssh-user

# 密码只放在本机覆盖文件里，不要提交。
VPN_PASSWORD=your_vpn_password

# 临时覆盖 Linux 分流脚本。
VPN_SCRIPT_CMD="/usr/local/bin/vpn-slice 10.0.0.0/8"
```

推荐用法：

- `config.toml` 保存稳定的非密码配置。
- `.env` 或 `ENV_FILE` 保存密码、本地临时覆盖、私密值或调试值。
- 不要把真实 VPN 用户名、密码、内网主机名或端口提交到仓库。

## 常见配置问题

### 提示“未设置 VPN_HOST”或“未设置 VPN_USER”

确认 `~/.config/och/config.toml` 存在，并且 `[vpn].host` 和 `[vpn].user` 非空。若使用 `ENV_FILE` 或 `.env` 覆盖，也检查里面是否把它们覆盖成空值。

### `och` 未传目标时报错

确认 `[ssh].host` 已设置，例如：

```toml
[ssh]
host = "och-target"
```

也可以直接传目标：

```bash
och och-target
```

### Linux 提示需要 `VPN_ROUTES`

在 TOML 中填写：

```toml
[routes]
extra = ["10.0.0.0/8"]
```

或者在 env 覆盖文件中设置：

```bash
VPN_ROUTES="10.0.0.0/8"
```

### GUI 保存后 SSH 没走 OCH

配置文件只负责字段；SSH 自动代理还需要 `~/.ssh/config` 包含：

```sshconfig
Include ~/.ssh/och.config
```

可以在 GUI 的 SSH 页面点击“安装 Include”。
