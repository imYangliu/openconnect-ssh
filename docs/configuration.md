# OCH 配置文件说明

OCH 的非敏感配置只有一份 TOML：

```text
~/.config/och/config.toml
```

可以用 `OCH_CONFIG_FILE` 指向另一份 TOML。除 `VPN_PASSWORD` 和 `OCH_SECRETS_FILE` 外，CLI 不从环境变量读取普通配置，也不会读取当前目录的覆盖文件。

## Secret 存储

VPN 密码不要写入 TOML。

- CLI `och setup`：启动 Rust 全屏 TUI 向导，并写入 `~/.config/och/secrets.env`；文件权限必须是 `0600`，macOS 和 Linux/Debian 相同。SSH 可以跳过。
- macOS GUI：可以在启用保存时写入 Keychain service `och`。
- 临时运行：可以直接传 `VPN_PASSWORD=...`，优先级高于 secret 文件和 macOS Keychain fallback。

CLI secret 文件只允许这一项：

```bash
VPN_PASSWORD="your-vpn-password"
```

可以用 `OCH_SECRETS_FILE=/path/to/secrets.env` 指向其他 secret 文件；该文件同样必须是 `0600`。

## 完整示例

```toml
# Generated by OCH. VPN password is stored separately, not in this file.

[vpn]
host = "vpn.example.com"
user = "your_vpn_username"
auth_group = ""

[ssh]
host = "och-target"
target_host = "your-campus-host.example"
user = "your-ssh-user"
port = "22"

[routes]
mode = "openconnect"
extra = ["10.0.0.0/8", "192.168.0.0/16"]

[dns]
mode = "openconnect"

[paths]
# Runtime helper paths are fixed by the installed app or CLI layout.

[app]
language = "system"
```

## 字段说明

### `[vpn]`

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `host` | 是 | VPN 网关地址，传给 `openconnect`。 |
| `user` | 是 | VPN 用户名。 |
| `auth_group` | 否 | 认证组/realm；网关不需要时留空。 |

### `[ssh]`

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `host` | 是 | 托管 SSH Host alias，例如 `och-target`；用户日常通过 `ssh <host>` 使用。 |
| `target_host` | 是 | 真实目标主机；也用于连接后验证路由和端口。 |
| `user` | 否 | SSH 用户；默认是当前系统用户。 |
| `port` | 否 | SSH 端口；默认 `22`。 |

GUI、`och setup` 和 `och tui` 会在这些字段完整且 SSH 启用时生成 `~/.ssh/och.config` 中的托管 Host。跳过 SSH 时不会写入或删除旧 managed SSH 文件。旧 `src/och-setup.sh` 仅保留为 legacy compatibility。

### `[routes]`

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `mode` | 否 | `openconnect` 或 `extra`；默认 `openconnect`，保持 OpenConnect 原生路由行为。旧配置缺少 `mode` 但 `extra` 非空时兼容为 `extra`。 |
| `extra` | 否 | CIDR 数组，例如 `["10.0.0.0/8"]`；仅在 `mode = "extra"` 时额外加入 VPN tunnel。 |

macOS 默认不接管 OpenConnect 路由。只有 `mode = "extra"` 且 `extra` 非空时，OCH 才会通过内置 route wrapper 把这些 CIDR 加到 OpenConnect tunnel。`extra` 不是直连绕过规则。

Linux/Debian 不再自动选择第三方分流脚本。需要额外路由时，请使用系统网络策略、服务端下发路由，或在部署层显式配置 OpenConnect 脚本。

### `[dns]`

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `mode` | 否 | `openconnect` 或 `ignore`；默认 `openconnect`，允许 OpenConnect/vpnc-script 按服务端下发 DNS 更新系统 DNS。`ignore` 会在 macOS wrapper 中忽略下发 DNS，保留现有系统 DNS。 |

`ignore` 目前通过 macOS 内置 wrapper 生效；Linux/Debian 上请用系统网络策略或自定义 OpenConnect 脚本控制 DNS。

### `[proxy]`

可选 section。默认配置不会写入 `[proxy]`；只有在 GUI 中启用 Proxy，或手动添加这个 section 时，OCH 才会保留这些反向端口映射字段。当前 Rust CLI 主路径是 `ssh och-target`，不依赖 `[proxy]`。

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `local_host` | 否 | 反向端口映射的本地地址，启用后默认 `127.0.0.1`；保留给 GUI/旧 wrapper 兼容或后续能力。 |
| `local_port` | 否 | 反向端口映射的本地端口，默认 `7890`。 |
| `remote_port` | 否 | 反向端口映射的远端端口，默认 `7890`。 |

### `[paths]`

保留为空 section，表示 helper 路径由安装布局决定。不要在这里写 `och`、`och_vpn` 或 `askpass`；严格解析会拒绝这些字段。

### `[app]`

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `language` | 否 | `system`、`en`、`zh-Hans` 或 `zh-Hant`；默认 `system`。 |

## 严格解析

`config.toml` 是严格 schema：

- 未知 section 或未知 key 会报错。
- `[paths]` 只能保留为空 section。
- 必填字段缺失会报错。
- 普通配置不能用环境变量覆盖。

## 排障

### 提示缺少 `[vpn].host` 或 `[vpn].user`

确认 `OCH_CONFIG_FILE` 指向的 TOML 存在，并且 `[vpn]` 里填写了 `host` 和 `user`。

### 提示 secret 文件权限不安全

执行：

```bash
chmod 600 ~/.config/och/secrets.env
```

### 路由没有按预期进入 VPN

macOS 检查 `[routes].mode` 是否为 `extra`，以及 `[routes].extra` 是否包含目标 CIDR。Linux/Debian 检查 OpenConnect 服务端下发路由或你的系统级路由配置。
